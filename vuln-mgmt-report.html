Program Overview

Inception to completion

This project simulates the implementation of a comprehensive vulnerability management program, from an organization with no existing policy or practices in place, to a formal policy enacted, stakeholder buy-in secured, and a full cycle of organization-wide vulnerability remediation successfully completed.

Technology used: Tenable (enterprise vulnerability management platform), Azure Virtual Machines (Nessus scan engine + scan targets), and PowerShell & Bash remediation scripts.

Building the Program

Policy, buy-in, and sign-off

01
Policy Draft

Starting from nothing

Drafted a Vulnerability Management Policy outlining scope, responsibilities, and remediation timelines — the starting point for stakeholder engagement.

02
Stakeholder Buy-In

Negotiating the real world

A mock meeting with the server team assessed their capability to meet remediation timelines. Feedback led to extending the critical remediation window from 48 hours to one week — a realistic, collaborative compromise.

03
Leadership Sign-Off

Making it official

After revisions addressing aggressive timelines, the policy received final approval from senior leadership, giving it the authority to guide the program and resolve pushback.

04
Scan Permission

Earning trust one server at a time

Negotiated with the server team to scan a single server first, monitoring resource impact, using just-in-time Active Directory credentials for secure, controlled access.

Assessment & Prioritization

What got fixed first, and why

After the initial authenticated scan, vulnerabilities were assessed and prioritized by ease of remediation and impact:

Priority 1

Third-party software removal (outdated Wireshark)

Priority 2

Secure configuration — insecure protocols & cipher suites

Priority 3

Secure configuration — Guest account group membership

Priority 4

Windows OS updates

Remediation packages and scan reports were distributed to the server team, reviewed in a follow-up meeting, and submitted to the Change Control Board (CAB) — approved with a rollback script and a tiered deployment approach.

First Cycle Remediation Summary

The remediation process reduced total vulnerabilities by 80%, from 30 to 6. Critical vulnerabilities were resolved by the second scan (100%). High-severity vulnerabilities dropped by 90%. Mediums were reduced by 76%. In a production environment, asset criticality would further guide future remediation priorities.

Maintenance Mode

What happens after the first cycle

After completing the initial remediation cycle, the program transitions into ongoing maintenance — ensuring vulnerabilities continue to be managed proactively rather than treated as a one-time project.

Scheduled Scans

Regular scans (weekly or monthly) to detect new vulnerabilities as systems evolve.

Patch Management

Continuous application of security patches — no critical vulnerability left unpatched.

Remediation Follow-ups

New vulnerabilities addressed promptly, prioritized by risk and impact.

Audit & Compliance

Internal audits ensure alignment with the vulnerability management policy and external regulations.
