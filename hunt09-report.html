PRIVILEGE ESCALATION

The Four-Step Climb_

STEP 1

Kerberoast

p.singh requested a TGS for the svc_backup SPN — RC4 (0x17) encrypted, crackable offline. T1558.003

STEP 2

Offline Crack

The RC4 ticket cracked offline, recovering svc_backup's plaintext password.

STEP 3

DCSync

svc_backup's replication rights pulled every NT hash from NTDS — including d.williams and KRBTGT. T1003.006

STEP 4

Pass-the-Hash

d.williams' NT hash authenticated Domain Admin sessions across every estate host. T1550.002

⚡ CLINIC KEY

Kerberoasting is low-noise — any domain user can request a TGS with no elevated rights. The only detection window before DCSync is EventID 4769 with EncryptionType 0x17 (RC4). If svc_backup had been configured AES-only, the entire chain collapses at Step 2.

LATERAL MOVEMENT

wmiexec (Impacket)_

The attacker used Impacket's wmiexec to execute commands remotely on GF-SRV01, GF-BKP01, and GF-DC01 — a semi-interactive shell via WMI that leaves a distinctive forensic signature.

DETECTION GAPS

What the Logging Missed_

Kerberoasting Alerting

EventID 4769 existed but no alert fired on EncryptionType 0x17. Fix: alert on 4769/RC4 for non-machine accounts; enforce AES-only on service accounts.

AD Object Auditing (SACL)

EventID 5136/4662 never fired — no SACLs configured. Fix: enable Directory Service Access auditing on domain objects and policy containers.
