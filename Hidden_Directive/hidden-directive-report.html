ATTACK CHAIN

One Day, Foothold to Domain_

10:15

Initial Access

The AI page-summariser (python.exe) fetched an attacker page and was command-injected into a hidden, AMSI-bypassing PowerShell loader → AdaptixC2.

11:49

Persistence & SYSTEM

sancadmin planted a scheduled task, a Run key, and a hijacked service (GFUpdaterSvc) — all executing as SYSTEM.

12:00

Lateral → SRV01

A svc_backup password spray landed a Type-3 logon onto the file/SQL server from the workstation.

15:47

Domain Compromise

svc_backup reached the DC; LSA secrets, SAM and NTDS were dumped and a Kerberos Golden Ticket was forged.

19:34

Exfiltration

Finance and domain data staged, compressed to exfil.zip, and uploaded over PowerShell to an attacker-controlled Azure Blob.

⚡ NOVEL ATTACK VECTOR

Initial access was neither a phish nor a CVE — it was a trusted internal AI tool. Greenfield’s page-summariser fetches external web content on a schedule; the attacker served it a poisoned page whose contents were treated as instructions, driving the summariser’s python.exe to spawn a PowerShell loader. Any automation that ingests untrusted external content is an attack surface most organisations haven’t modelled yet.

METHOD · TWO WORKSPACES

Build the Spine, Confirm the Crown Jewels_

The evidence lived across two SIEMs, and neither told the whole story. Process, auth and network flows came from LAW-SilentCorridor (ASIM); LSASS access, NTDS/DCSync, the Golden Ticket and Defender’s technique labels came from LAW-Cyber-Range (Defender XDR / MDE). The entry point was proven by pivoting to the parent of the first malicious PowerShell.

Signature find — the entry vector

T1059.001
// LAW-SilentCorridor · what did the AI summariser spawn?
WindowsProcess_CL
| where DvcHostname has "GF-WS01"
| where ActingProcessFilePath has "python.exe"
| project TimeGenerated, ActorUsername, TargetProcessCommandLine
| sort by TimeGenerated asc
// -> powershell -NoP -W Hidden -Exec Bypass : AMSI bypass + DownloadData(cloud-endpoint.net) + XOR 0x4A shellcode inject

Parent process python.exe (the summariser) launching a hidden, execution-policy-bypassing PowerShell is the whole case in one row — corroborated by network telemetry to cdn.cloud-endpoint.net seconds later.

Handover detection — catch it next time

T1566 / T1059
// Any automation / LLM tool spawning a shell
WindowsProcess_CL
| where ActingProcessFilePath has_any ("python.exe","node.exe")
| where TargetProcessFilePath has_any ("powershell","cmd.exe","pwsh")
| project TimeGenerated, DvcHostname, ActorUsername, ActingProcessFilePath, TargetProcessCommandLine

Shipped to the defender alongside spray-landing (fail→success) and unsanctioned-cloud-egress rules, so the entry vector can’t recur silently.
