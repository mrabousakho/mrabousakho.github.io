KQL FIELD NOTES

Microsoft Sentinel & Defender_

The queries that show up in almost every dual-SIEM hunt.

LSASS Memory Access Hunt

T1003.001
DeviceProcessEvents
| where FileName in~ ("procdump.exe","rundll32.exe","taskmgr.exe")
| where ProcessCommandLine has "lsass"
| project Timestamp, DeviceName, AccountName, ProcessCommandLine
| order by Timestamp desc

Used to catch credential-dumping attempts against LSASS — the same pattern that surfaced the DCSync activity in the Rocky Clinic breach.

DCSync / Replication Abuse

T1003.006
SecurityEvent
| where EventID == 4662
| where Properties has "1131f6ad-9c07-11d1-f79f-00c04fc2dcd2"
| summarize Attempts = count() by Account, Computer
| where Attempts > 0

Flags accounts requesting directory replication rights they shouldn't have — the core detection behind the MTS-DC domain controller compromise pattern.

Rare Parent/Child Process Pairs

Persistence
DeviceProcessEvents
| summarize Count = dcount(DeviceName) by FileName, InitiatingProcessFileName
| where Count < 3
| order by Count asc

Surfaces uncommon process lineage — the same "rare parents" logic used to catch persistence scripts hiding as monitoring services.

Linux MDE Process Timeline

Field Note
DeviceProcessEvents
| where DeviceName == "rocky83"
| project Timestamp, InitiatingProcessFileName, FileName, ProcessCommandLine
| order by Timestamp asc

Key gotcha: on Linux MDE telemetry, Timestamp is the primary time field — not TimeGenerated. Cost me time on Hunt 07 until I caught it.

SPL FIELD NOTES

Splunk_

What I reach for on the ELK/Splunk side of a dual-SIEM case.

Brute Force / Failed Logon Spikes

T1110
index=auth sourcetype=sshd "Failed password"
| stats count by src_ip, dest_ip
| where count > 20
| sort -count

First pass on any SSH-facing box — this is what caught the botnet pattern in the XorDDoS Linux cryptominer hunt.

C2 Beaconing Interval Check

Command & Control
index=network dest_port=443
| bin _time span=5m
| stats count by src_ip, dest_ip, _time
| eventstats stdev(count) as jitter by src_ip, dest_ip
| where jitter < 2

Looks for suspiciously regular connection intervals — low jitter is the tell for scripted C2 rather than normal human traffic.

AUTOMATION NOTES

Gotchas From Building the Agent Pipeline_

Python API timespan: calling Sentinel's API from Python requires an explicit timespan=timedelta(days=9999) or queries silently scope to a default window and miss older evidence.

SSL errors: cross-platform API calls to Azure endpoints need certifi installed, or you'll hit certificate verification failures that look like a networking problem but aren't.

dcount() over count(): for "how many unique hosts/users touched this" questions, dcount() is the right call — count() will overcount repeated events from the same entity.
