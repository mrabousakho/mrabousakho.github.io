Why two channels? Resilience — one survives if the other is cut — plus detection splitting: neither channel alone crosses the alert threshold.

CREDENTIAL THEFT

The Three-Step LSASS Chain_

🔒 HIGH-FIDELITY INDICATOR

Step 1: OpenProcessApiCall (0x1410, PROCESS_QUERY_INFORMATION — a probe). Step 2: OpenProcessApiCall (0x1F0FFF, PROCESS_ALL_ACCESS — full unrestricted). Step 3: ReadProcessMemoryApiCall — confirmed memory read, credentials dumped. PROCESS_ALL_ACCESS on lsass.exe from a non-system PowerShell process is one of the highest-fidelity credential theft indicators in Windows telemetry.

KQL_PLAYBOOK

Operator Signature Flags Hunt_

Query 02 · Operator Signature Flags

Detection
DeviceProcessEvents
| where DeviceName == "azwks-phtg-01"
| where FileName =~ "powershell.exe"
| where ProcessCommandLine has_any (
    "-WindowStyle Hidden", "-ExecutionPolicy Bypass",
    "-EncodedCommand", "-NoProfile")
| project TimeGenerated, ProcessCommandLine, InitiatingProcessFileName
| order by TimeGenerated asc

Any PowerShell process carrying all three signature flags is operator tooling — legitimate software never configures persistence this way.
