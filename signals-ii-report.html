Executive Summary

Maintenance that wasn't

This hunt reconstructs operator activity on the PHTG estate following the confirmed initial access established in Hunt 05. The operator's activity consistently mimicked legitimate HealthCloud administration — nearly all of it looked like routine maintenance. The value of this hunt is identifying deliberate intent hidden inside ordinary-looking telemetry.

The operator entered via credential reuse (not brute force), moved laterally to a second host, deployed a full persistence framework masquerading as HealthCloud services, established dual C2 channels, tampered with Windows Defender, and concluded with a confirmed LSASS memory read — credential theft proven by telemetry.

First Footsteps

Why the script was named "_.ps1"

first operator script · 10:11:42
powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\Users\vmAdminUsername\Documents\PHTG\_.ps1"
Why "_.ps1"?

An underscore sorts to the bottom of directory listings — designed to be overlooked during casual inspection.

Three scripts launched in rapid succession, each carrying the same three signature flags: -NoProfile -WindowStyle Hidden -ExecutionPolicy Bypass.

The Beacon Pair

Two parallel C2 mechanisms

ChannelMechanismBenefit
PHTGHealthCloudSvc.exe /healthcheckLOLBin masquerade (claims OriginalFileName = bitsadmin.exe)22 executions — blends with service health checks
Invoke-WebRequest -EncodedCommandBase64-obfuscated PowerShell beaconsEvades simple string detection

Why two channels? Resilience — one survives if the other is cut — plus detection splitting: neither channel alone crosses the alert threshold.

Anti-Forensics

An exclusion that existed for seconds

temporary exclusion pattern
Add-MpPreference -ExclusionPath "C:\Users\vmAdminUsername\Documents\PHTG"
# ... execute _.ps1 ...
Remove-MpPreference -ExclusionPath "C:\Users\vmAdminUsername\Documents\PHTG"

The Defender exclusion existed only for seconds — no permanent registry trace left behind. This is sophisticated anti-forensics, not a careless operator.

Credential Theft — LSASS

The three-step chain

StepActionTypeDesiredAccessMeaning
1OpenProcessApiCall0x1410PROCESS_QUERY_INFORMATION — low privilege probe
2OpenProcessApiCall0x1F0FFFPROCESS_ALL_ACCESS — full unrestricted
3ReadProcessMemoryApiCall—CONFIRMED: memory read, credentials dumped
High-Fidelity Indicator

PROCESS_ALL_ACCESS (0x1F0FFF) on lsass.exe from a non-system PowerShell process is one of the highest-fidelity credential theft indicators in Windows telemetry.

Detection Playbook

Operator signature flags hunt

KQL · query 02
DeviceProcessEvents
| where DeviceName == "azwks-phtg-01"
| where FileName =~ "powershell.exe"
| where ProcessCommandLine has_any (
    "-WindowStyle Hidden", "-ExecutionPolicy Bypass",
    "-EncodedCommand", "-NoProfile")
| project TimeGenerated, ProcessCommandLine, InitiatingProcessFileName
| order by TimeGenerated asc

Any PowerShell process in a Run key or Startup entry carrying all three signature flags is operator tooling — legitimate software never configures persistence this way.
